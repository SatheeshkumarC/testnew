VSM - Application to Cloud Analyzer

Assessment Summary

Technology

Primary Language - Primary Language contains the list of programming languages used in the application such as Java, .Net etc,...

Secondary Language - Secondary Language contains the list of languages used in the application that are not listed in the Primary Language like CSS, JSP ,HTML etc,...

App Server - App Server contains the list of all the application servers used in the application like Weblogic, Websphere, Tomcat, JBoss etc,...

Primary Framework - Primary Framework contains the list of primary frameworks used in the application like Spring, Struts , Hibernate etc,...

Secondary Framework - Secondary Framework contains the list of secondary frameworks used in the application like Play, Grails etc,...

Database - Database contains the list of all the databases used in the application like MySQL, Oracle, SQL Server etc,...

Authentication - Authentication contains the list of the authentication techniques used in the application like Spring security etc,..

COTS Products - COTS Products contain the list of all the vendor or third-party software that are used by the application like Salesforce, OpenCMS etc,...

Libraries - Libraries contain the list of all the libraries used in the application like Apache commons-logging, Google GSON etc,...

General

Lines of Code - Lines of Code contains the total number of lines of code in the application. Moreover, it is subdivided based on the programming languages along with the number of lines of code corresponding to each programming language.

App Classes - Classes correspond to the number of app classes used in the application.

Test Cases - Test cases denote the number of test cases written for the application like Junit test cases, TestNG test cases etc,...

Cloud Suitability

Cloud Suitability - Cloud Suitability denotes the suitability of the application to fit into the cloud in percentage

Migration Complexity

Migration Complexity - Migration Complexity represents the complexity level in migrating the application to cloud. It is categorised as Low, Medium and High.

Migration Complexity Points - Migration Complexity Points denote the estimated effort needed in points to migrate the application to cloud.

Code Modularity - Code Modularity rating is calculated based on the number of lines of code, Cohesion and Coupling values.

Code Complexity - Code Complexity denotes the complexity of the application code.

External Interfaces/ Dependencies

External Interfaces/ Dependencies - External Interfaces/ Dependencies contain the list of external dependencies used by the application like SMTP ,LDAP etc,..

Migration Strategy Recommmendations

Rehost

Rehost follows the Lift and Shift strategy

Replatform

Replatform requires few changes to be made to the code before migrating it to the cloud

Refactor

Refactor Strategy needs the code to be modified before migrating it to the cloud

Rebuild

Rebuild Strategy needs the entire application to be rebuilt since the existing codebase will not fit into the cloud

Cloud Anti-Patterns Analysis

Anti-Patterns Summary

Anti-Patterns Summary - Anti-Patterns Summary gives the summary of the anti-patterns found in the application and lists them based on their type and sub type. A short description of the detected anti-pattern along with the recommendation is also provided. Total anti-patterns correspond to the number of usages of each anti-pattern in the application

Anti-Patterns Detail

Anti-Patterns Detail - Anti-Patterns Detail gives the complete detail of the anti-patterns listed in the Anti_Patterns Summary along with the location details of all the anti-patterns

Known Issues

Known Issues - Known Issues gives the list of all known issues like exception handling, log files etc., that are used in the application

Cloud Suitability Checklist

Cloud Suitability Checklist

Cloud Suitability Checklist lists the various standards with respect to cloud. It then classifies the application based on each standard as low, medium, high or not applicable
